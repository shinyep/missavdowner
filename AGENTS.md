# 项目规则

## 打包与发布

- 除非用户明确要求“提交 release”或“发布 release”，否则只允许执行编译、构建、打包，不得擅自创建、提交或更新 release。
- 每次 `main` 分支有代码改动时，若要进行打包或交付，必须同步更新版本号，并添加对应的版本更新说明。

## 图站规则接入

- 新增图站前，先抓取目标页面 HTML，确认 DOM / JSON-LD / 分页、图片加载方式和正文边界，再判断走“专用解析器”还是“通用提取器”，并优先选择专用解析器。
- 先确认接入范围：整站通用规则、仅部分路径支持，还是单链接特判；只要站点结构一致，就做成站点识别 + 专用解析器，而不是针对单链接硬编码。
- 设计先写 spec：`docs/superpowers/specs/YYYY-MM-DD-<site>-design.md`，覆盖站点识别、标题提取优先级、正文图片提取范围、分页策略、兜底逻辑、错误处理、测试与验收标准。
- 实现先建隔离分支：默认使用 `codex/<site>-gallery-rule`，不直接在 `main` 上改；如用户要求在当前分支实现，则按要求执行。
- 测试优先：先写失败测试 `python/tests/test_<site>_parser.py`，使用固定 HTML / JSON-LD 样本，不依赖实时网络；先跑红灯，再补最小生产代码；通过后做回归验证。
- 代码接入点固定为 `python/image_crawler.py`：新增 `_is_<site>_url()`、`_parse_<site>_gallery()`，并在 `parse_gallery()` 里完成路由；优先只抓正文容器或结构化数据，避免混入相关推荐、侧栏、广告和装饰图。
- 收尾时补齐支持站点文档：若 `README.md` 中有站点列表，就把新站点一并更新。
- 合并前至少完成：站点识别、标题提取、正文图片提取、分页/兜底、失败测试、必要文档更新；如果主 `main` 有代码改动且要打包，再按版本规则更新版本号。
- 常见陷阱优先写进 spec：例如站名标题和正文标题不同、首页 `h1` 可能只是站点名、分页 URL 规则、正文容器选择器变更、图片 `data-src` / `src` 差异。
- 临时调研产物只留结论，不留脚本；如生成过临时抓取脚本，完成后必须删除。
### 常用实现模式

- “正文 DOM 优先”模式：先定位正文容器（如 `.entry-content`、`article`、`.elementor-widget-theme-post-content`），只从容器内提取图片；站点名和文章标题容易冲突时，优先取 `meta og:title` / `title` / 正文 `h1`，不要直接把站点 `h1` 当文章标题。
- “结构化数据优先”模式：若页面有 `ImageGallery` / 图集 JSON-LD，优先从 `name` 取标题，从 `itemListElement` 取图片，从 `pagination` 取分页；当前页 JSON-LD 不完整时，再退回正文 `img[data-src]` / `img[src]` 兜底。
- 分页接入时，先写清楚分页 URL 规则（如 `/page/N/`、`?page=N`、`a.page-next`），并在测试里覆盖：首页能提取、后续页能遍历、缺字段时能兜底。
- 标题解析验收标准统一为“能还原文章真实标题”，而不是只返回站点名或站点后缀；若站点存在多层标题，需要把优先级写进 spec 和测试。
- 图片过滤要优先明确排除项：推荐区、侧栏、广告、logo、icon、avatar、banner、像素图、带 `?w=...&ssl=1` 的 CDN 缩略图、`data:` URL。
- 验收时至少覆盖两类样本：一个是正常样本，一个是“标题被站点名污染 / 首页结构变化 / 结构化数据缺失”的兜底样本；两条路径都应有测试。
- 每次新增图站后，顺手更新：`README.md` 的站点列表、spec、回归测试；不要只改生产代码而不补文档和测试。

### 工具与执行规范
- 调研阶段优先用联网抓取工具获取真实页面 HTML，再用文件读取工具整理样本；避免凭空猜页面结构。
- 生成临时调研脚本时，优先用代码执行工具验证；验证完成后，用文件删除工具清理临时脚本与临时页面文件，只保留 spec、测试和生产代码。
- 修改生产代码和测试时，优先用代码编辑工具进行定点修改；不要用一次性脚本覆盖整个文件。
- 提交代码前，优先用 shell 或项目脚本跑本地回归测试；如果当前环境无法运行测试，必须明确告知用户，而不能默认跳过。
- 更新文档时，优先用文档编辑工具精确修改 AGENTS.md、README.md、spec 等，不要重复生成多个同名临时文档。
- 使用搜索工具定位现有解析器、识别函数、标题提取函数、图片提取函数和测试文件；避免重复新建同类代码路径。
